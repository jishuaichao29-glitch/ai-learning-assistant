## 1. 🤖 自动化 CI/CD 流水线落地

* **考核项对标**：项目实现了CI/CD（如GitHub Actions自动部署/集成）。
* **工程化实现**：
    本项目在根目录下配置了标准的 GitHub Actions 工作流。每次开发者执行 `git push` 向主分支提交代码时，均会触发云端虚拟服务器的自动化构建流水线。
* **云端自动化步骤**：
    1.  自动启动最新的 Ubuntu 虚拟操作系统环境。
    2.  无缝检出（Checkout）项目最新主干源码。
    3.  自动配置标准的 Python 3.10 云端运行时环境。
    4.  自动化安装测试所需的轻量化核心依赖（`pytest`、`flask`、`flask-cors`）。
    5.  自动跨目录切入后端文件夹，拉起测试引擎执行断言。
* **核验路径**：
    * **核心配置文件**：项目根目录下的 `.github/workflows/python-tests.yml`。
    * **云端运行证明**：请直接点击本项目 GitHub 仓库顶部的 **`Actions`** 标签页。目前云端多次集成的历史流水线均已**全绿通过（✅ Passing）**，确保了代码在标准干净环境下的可移植性与健壮性。

## 3. 🛡️ 生产级工程化防御手段

本项目全面拒绝粗糙的逻辑堆砌，严格遵循现代化 Web 工程的生产标准，落地了三项硬核工程手段：

### 🔑 (1) 环境变量隔离与去敏
* **实现方案**：涉及大模型交互的 API Key、敏感密钥及数据库路径等生产资料，全部从业务代码中抽离，统一通过系统环境变量进行隔离读取。
* **安全规范**：项目根目录下提供了去敏的 **`backend/.env.example`** 模板文件，真实敏感密钥已全部剔除，完美符合公开仓库的安全开源规范，彻底杜绝密钥外泄风险。

### 📝 (2) 持久化 logging 日志系统
* **实现方案**：在后端核心逻辑 `backend/app.py` 中，全面初始化了 Python 标准 `logging` 日志组件。
* **运行效果**：系统运行期间的所有动态行为、接口请求以及业务流转，均会持久化输出至本地的 **`backend/app.log`** 文件中。这使得系统在无人值守的居家实训环境下，具备了行为可追溯与可审计的生产级特性。

### 🚨 (3) 高鲁棒性错误异常捕获与监控
* **实现方案**：全面摒弃了缺乏上下文堆栈的 `print` 报错方式。
* **异常监控**：在涉及流式聊天（`/api/chat`）及 RAG 知识库上传等高频易错的核心大模型接口中，部署了严密的 `try-except` 防御性代码。在捕获异常时，统一调用 `logger.error(..., exc_info=True)`。
* **技术价值**：该手段能在服务发生不可控崩溃（如网络超时、模型未响应）时，在 `app.log` 中自动捕获并落盘**最完整的 Traceback 错误堆栈信息**，极大提升了黑盒测试环境下的 Debug 效率与线上监控能力。
