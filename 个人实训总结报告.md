# 个人实训总结报告
## 一、遇到的最大困难及解决过程
在本次全栈项目开发中，我遇到的最大挑战是跨越前后端分离的鸿沟以及解决数据库双表关联带来的数据持久化问题。

项目初期，前后端联调时频繁出现跨域错误和端口占用问题。我通过在 Flask 中配置 flask-cors 并将后端服务绑定到 0.0.0.0 解决了跨域问题，使用 PowerShell 的端口强杀命令解决了进程占用问题。

最棘手的是实现点赞/点踩功能时的数据丢失 Bug。我设计了 LikedMessage 和 DislikedMessage 两张独立表来存储用户反馈，但在实现过程中遇到了三个关键问题：一是前端点击事件没有触发后端请求，二是后端数据库操作缺少 commit() 导致数据只存在于内存中，三是历史消息接口没有进行表连接查询，导致刷新后状态无法回显。

我利用 AI 作为调试助手，通过在关键代码处埋设日志，逐步定位问题所在。最终通过补齐前端 fetch 请求、添加 db.session.commit() 、使用 Left Join 重构查询逻辑，成功打通了全链路数据闭环。

## 二、对 AI 辅助编程的心得体会
通过这次实训，我深刻体会到 AI 不是简单的代码生成器，而是一位高级导师和合作伙伴。

在项目初期，我曾天真地认为只需输入"帮我做一个聊天系统"就能得到完美的代码，但实际效果并不理想。后来我学会了将复杂问题拆解为多个子任务，用清晰的工程化思维引导 AI 逐步实现。比如在实现 RAG 知识库时，我将任务分解为 PDF 上传、文本切片、向量存储、检索增强四个步骤，每一步都给出明确的技术要求和约束条件。

AI 在处理重复性工作和提供技术方案方面表现出色，但最终的架构设计和问题解决仍需要开发者的工程化思维。我学会了如何正确"提问"——给出明确的输入输出要求、提供上下文信息、指定技术约束，才能获得高质量的代码输出。

## 三、个人在项目中的具体贡献
在本次实训项目中，我独立承担了全栈开发工作，具体贡献如下：

前端开发方面 ：基于 Next.js 14 + React 18 + Tailwind CSS 构建了完整的用户界面，包括高颜值首页、聊天室交互大厅、个人中心数据看板等页面。实现了沉浸式宽屏收纳模式，通过 Tailwind CSS 的过渡动画实现了侧边栏的丝滑折叠/展开。开发了划词悬浮追问功能，通过全局事件监听和原生内联样式定位，实现了精准的微交互效果。

后端开发方面 ：基于 Flask 3.0 开发了完整的后端服务，共实现了 26 个 API 接口，涵盖用户认证、会话管理、聊天功能、消息历史、用户管理、统计分析、收藏功能、知识库和搜索功能等模块。搭建了基于 LangChain + FAISS + Sentence Transformers 的 RAG 知识库系统，支持 PDF 文档上传、文本切片、向量化和相似性检索。

工程文档方面 ：编写了完整的 README.md 项目文档，包含项目介绍、技术栈、安装运行指南、API 接口文档和数据库表结构说明。整理了详细的 prompt_log.md 开发日志，记录了从项目初始化到功能实现的全过程。

## 总结
通过本次实训，我不仅掌握了前后端分离的开发模式和现代 Web 技术栈，更重要的是培养了工程化思维和问题解决能力。AI 辅助编程确实能大幅提升开发效率，但开发者的主动思考和架构设计能力依然是核心竞争力。未来我将继续深入学习全栈开发技术，不断提升自己的工程实践水平。
